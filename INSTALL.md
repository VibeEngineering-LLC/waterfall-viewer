## Что в итоге получится

Десктоп-окно с двумя вкладками:

- **3D Waterfall** — поверхность Время × Энергия × Отсчёты (OpenGL): вращение мышью,
  зум колесом, панорама средней кнопкой.
- **2D Карта (Время×Энергия)** — двумерная карта с лог-контрастом и прямоугольной
  выборкой (ROI). Справа — панель срезов: спектр выбранного окна времени, временной
  ряд интенсивности в полосе энергий и суммарные отсчёты в прямоугольнике.

Суточные записи (десятки тысяч временных срезов) грузятся потоково и рендерятся через
LOD-прорежку — без верхнего предела на длину записи.

## Требования к системе

- **ОС:** Windows 10/11 (разрабатывалось и проверялось на Windows 11).
- **Python:** 3.12 или 3.14 (64-bit). Проверено на 3.14.5.
- **Видео:** GPU/драйвер с поддержкой **OpenGL 2.1+** (для 3D-вкладки). Подходит почти
  любая встроенная или дискретная видеокарта последних 10 лет.
- **Память:** от 4 ГБ; для суточных записей — больше (загрузчик потоковый, но матрица
  отсчётов держится в RAM).
- **Место на диске:** ~600 МБ под зависимости (основной объём — PySide6/Qt).

## Шаг 1: Установить Python

Если Python уже стоит — пропусти шаг (проверь: `py -3.14 --version` или
`python --version` в PowerShell).

1. Скачай установщик с [python.org/downloads](https://www.python.org/downloads/)
   (версия 3.12 или 3.14, 64-bit).
2. При установке **обязательно** поставь галочку **«Add python.exe to PATH»**.
3. Проверь в PowerShell:
   ```powershell
   py -3.14 --version
   ```
   Должно вывести `Python 3.14.x`.

---

## Шаг 2: Получить файлы программы

Через git:
```powershell
git clone <URL-репозитория> atomspectra-waterfall-viewer
cd atomspectra-waterfall-viewer
```

Или скачай ZIP-архив репозитория и распакуй в любую папку.

## Шаг 3: Установить зависимости

Зависимости перечислены в [`requirements.txt`](requirements.txt):
`numpy`, `lxml`, `PySide6` (Qt), `pyqtgraph` + `PyOpenGL`, `pytest`.

### Путь А — venv (изолированно, рекомендуется)

```powershell
cd "путь\к\atomspectra-waterfall-viewer"
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
```

> **Важно.** Создавай venv тем же Python, которым потом будешь запускать. **Нельзя**
> пересоздавать venv поверх существующего другим интерпретатором (например, был 3.12,
> создал заново 3.14): `python -m venv` подменит `python.exe`, но **не** переустановит
> пакеты — получишь смешанные бинарники (`.pyd` от 3.12 под интерпретатором 3.14) и
> ошибку `ModuleNotFoundError: numpy._core._multiarray_umath`. Лечится удалением `.venv`
> и установкой заново. См. [`KNOWN_ISSUES.md`](docs/dev/KNOWN_ISSUES.md).

### Путь Б — глобально (без venv)

```powershell
py -3.14 -m pip install -r requirements.txt
```

`run.bat` сам выберет: при наличии `.venv` запустит из него, иначе — глобальным `py -3.14`.

## Шаг 4: Запуск

```powershell
.\run.bat                                   # пустое окно, файл открыть через меню
.\run.bat "C:\путь\waterfall (7).n42"       # сразу открыть файл
```

или напрямую модулем:
```powershell
py -3.14 -m awf "C:\путь\к\файлу.n42"
```

---

## Шаг 5: Открыть спектрограмму и проверить

1. **Файл → Открыть…** (Ctrl+O), выбери файл: AtomSpectra `.aswf`, RadiaCode `.rcspg`
   или N42 `.n42`/`.xml`.
2. В статус-строке появится «всего отсчётов …» — файл загружен.
3. **Вкладка 3D Waterfall:** зажми **левую кнопку** мыши и веди — поверхность вращается;
   **колесо** — зум; **средняя кнопка** — панорама.
4. **Вкладка 2D Карта:** потяни жёлтый прямоугольник — панель срезов справа обновляется
   вживую (спектр окна + временной ряд полосы + сумма отсчётов).

## Тесты

Слой данных покрыт автотестами (опорные значения — независимый оракул из байтов файла):

```powershell
.venv\Scripts\python.exe -m pytest -q      # из venv
py -3.14 -m pytest -q                       # глобально
```

38 тестов: ISO-длительности, CountedZeroes (hand-cases + fuzz vec≡scalar), полиномиальная
калибровка и обратное преобразование, загрузка образца, лимит срезов, аналитические
примитивы; загрузчики AtomSpectra `.aswf` и RadiaCode `.rcspg` (форма/dtype, значения,
калибровка, временные оси, t0, граничные случаи — на синтетических файлах); библиотека
нуклидов (парсер LSRM `.lib`, round-trip JSON).

> Тест загрузки N42-образца помечен `skipif` и **пропускается**, если в `sample_data/` нет
> файла `waterfall_sample.n42`. Сам образец **не входит в репозиторий** (N42 может содержать
> GPS/метаданные прибора, см. `.gitignore`) — положи свой `.n42` в `sample_data/` под этим
> именем, если хочешь прогнать и этот тест. Тесты загрузчиков `.aswf`/`.rcspg` строят
> синтетические файлы во временной папке и реального образца не требуют.

## Troubleshooting

| Симптом | Причина | Решение |
|---|---|---|
| `ModuleNotFoundError: numpy._core._multiarray_umath` | venv пересоздан другим Python поверх старого — смешанные бинарники | Удалить `.venv`, создать заново и `pip install -r requirements.txt` |
| `run.bat` пишет «is not recognized» / мусор в консоли | `.bat` запущен не из своей папки, либо правился в редакторе, добавившем не-ASCII в комментарии | Запускать из папки проекта; комментарии в `run.bat` держать ASCII-only |
| Открываются **два** окна | старая версия `run.bat` с fallback `|| python` | Обновить `run.bat` из репозитория (один запуск `py -3.14 -m awf %*`) |
| Чёрная 3D-вкладка / ошибка OpenGL | драйвер видео без OpenGL 2.1 | Обновить драйвер GPU; на ВМ включить 3D-ускорение |
| Окно открылось, но пустое | файл не открыт | **Файл → Открыть…**, выбрать `.aswf` / `.rcspg` / `.n42` |

Полный разбор известных проблем и совместимости — в [`KNOWN_ISSUES.md`](docs/dev/KNOWN_ISSUES.md).
