# Реестр задач внедрения — детальная спецификация

Подробный нумерованный backlog по промпту оператора (ТЗ 2026-06-25). **Последовательность не
фиксирована** — порядок решаем отдельно. Стратегия/фазы — в [`IMPLEMENTATION_PLAN.md`](IMPLEMENTATION_PLAN.md).

Каждая задача расписана по единому шаблону:
- **Цель** — что получает пользователь.
- **Подзадачи** — конкретные шаги реализации (адресуемые: `N.1`, `N.2`…).
- **API / сигнатуры** — предлагаемые функции/классы (помечено «предлагается»; источник-сигнатуры из
  SpectraVibe помечен «источник»).
- **Параметры / формулы** — точные значения, с провенансом `file:line` где портируем.
- **UI** — конкретные элементы интерфейса.
- **Тесты** — проверяемые критерии готовности.
- **Зависимости** — пакеты и задачи-предпосылки.
- **Объём** — S (≤полдня) / M (1–2 дня) / L (3+ дня), грубо.

Сложность данных: 🟢 чистый numpy/stdlib · 🟡 нужен scipy/sklearn · 🔵 только UI · 🟣 требует рефактора
модели. SpectraVibe (`gamma-spectrum-analysis`) — **референс на чтение**, пути `scripts/gamma/...`.

Сквозной инвариант реализации: код ≥25 строк → спецификация `scripts/ollama/_spec_<name>.md` →
кодоген Ollama `qwen3-coder:30b` → валидация (`py_compile`+pytest). Численное ядро — без Qt, в
`awf/analysis/` или `awf/model/`. Калибровка — ascending-полином (совместима с SpectraVibe).

# Группа II. Фон и пики (ТЗ-A.7, A.8, A.9)

## Задача 2 — Поиск пиков (Mariscotti + Currie) 🟢 · Объём M · *ТЗ-A.9*
**Цель:** автоматически находить фотопики в спектре среза/окна и в суммарном спектре.

**Подзадачи:**
- 2.1. Порт фильтра второй производной Гауссиана (Mariscotti) — свёртка спектра с ядром −G''.
- 2.2. Критерий значимости Currie L_C: пик принимается при `significance ≥ sigma_threshold`.
- 2.3. Banded segmentation: разбить спектр на полосы равного FWHM (max_ratio≈1.2), фильтр per-band.
- 2.4. Спайк-фильтр: отсев пиков с FWHM < 0.3×модели (узкие выбросы).
- 2.5. Предварительная площадь `area ≈ 2.507·σ·height`.
- 2.6. Интеграция: вызвать на `sum_spectrum`/`total_spectrum`, вернуть `list[FoundPeak]`.

**API (предлагается):**
```python
def find_peaks(counts_1d, fwhm_channels, *, sigma_threshold=3.0,
               spike_min_fwhm_frac=0.3) -> list[FoundPeak]
```
**Источник:** `peaks/search.py` `mariscotti_search()` (стр. 261–450); спайк-фильтр (413–431);
площадь (432); альт-метод `peaks/convolution_search.py` (matched-filter, для перекрёстной проверки).
**Параметры:** `sigma_threshold=3.0` (Currie 3σ), `max_ratio=1.2`, `spike_min_fwhm_frac=0.3`.
**UI:** маркеры пиков (`InfiniteLine`) на графике спектра в `SlicePanel`; чекбокс «Показать пики».
**Тесты:** синтетика 3 гауссианы (разной высоты) на наклонном фоне → находит ровно 3, центры ±0.5
канала, `significance` монотонна по высоте; узкий δ-выброс отсекается спайк-фильтром.
**Зависимости:** numpy. **Предпосылки:** 1; 8 (FWHM-модель; на старте — константный FWHM из заголовка).

## Задача 3 — Континуум/фон под пиком 🟢 · Объём M
**Цель:** оценить baseline под ROI пика для расчёта нетто.

**Подзадачи:**
- 3.1. Cowell: полином 1-го порядка по крыльям ROI (`numpy.polyfit`), интеграл baseline по full ROI.
- 3.2. Сглаженная ступень (Будыка 7.29): `B(E)=bg_r+(bg_l−bg_r)·0.5·erfc((E−E₀)/(σ√2))+slope·(E−E₀)`,
  `erfc` через `math.erfc` (без scipy).
- 3.3. Асимметричные окна фона: автосужение крыла при интерферирующем соседнем пике.
- 3.4. Выбор модели — параметр `method ∈ {'cowell','step'}`.

**API (предлагается):**
```python
def estimate_continuum(counts_1d, roi_lo, roi_hi, *, method='cowell',
                       wing=None) -> np.ndarray   # baseline по каналам ROI
```
**Источник:** `peaks/area.py:77–238` `cowell_area()`; `peaks/background_options.py:40–89`
`smoothed_step_bg()`; асимметрия `background_options.py:107–169`.
**Тесты:** на чисто-линейном фоне Cowell даёт нетто≈0; площадь известной гауссианы поверх фона в
пределах 3%; `step` непрерывна и монотонна между уровнями.
**Зависимости:** numpy, `math.erfc`. **Предпосылки:** 1.

**Подзадачи:**

- 5.2. Переключатель «Вычесть фон» в тулбаре; хранить и оригинал, и нетто (быстрое переключение).

## Задача 6 — Площадь пика и нетто-счёт 🟢 · Объём M
**Цель:** интегральная площадь/нетто пика с погрешностью (основа активности и MDA).

**Подзадачи:**
- 6.1. `cowell_area`: `S = N_gross − B`, `dS = √(N_gross + B)`.
- 6.2. (опц.) гаусс-фит площади — позже через scipy (Задача 23); сейчас numpy-Cowell.
- 6.3. Связать с найденными пиками: ROI = `peak.channel ± k·FWHM`.

**API (предлагается):**
```python
def peak_area(counts_1d, peak_channel, fwhm_channels, *, k_roi=2.5,
              continuum='cowell') -> PeakArea   # net, d_net, gross, baseline
```
**Источник:** `peaks/area.py` `cowell_area()`.
**Тесты:** площадь синтетической линии в пределах допуска; погрешность ~√(gross+B); ROI-границы.
**Зависимости:** numpy. **Предпосылки:** 2, 3.

## Задача 7 — MDA по ISO 11929 🟢 · Объём S
**Цель:** порог принятия решения, предел обнаружения и минимально детектируемая активность.

**Подзадачи:**
- 7.1. `L_C = k_α·√(2·n_bg)` (порог решения, counts над фоном).
- 7.2. `L_D ≈ k_α² + 2·k_α·√(n_bg + L_C²/4)` (предел обнаружения).
- 7.3. `A_MDA = L_D / (ε·I·t)` (в Бк), `k_α = 1.645` (α=5%, односторонний).

**API (предлагается):**
```python
def mda_for_peak(n_bg, live_time_s, efficiency, intensity_frac,
                 *, k_alpha=1.645) -> MdaResult   # L_C, L_D, A_MDA
```
**Источник:** `identification/mda.py:101–150` `mda_for_peak()`; константы (66–69).
**Тесты:** L_C/L_D/A_MDA совпадают с ручным расчётом ISO 11929 на заданных n_bg/ε/I/t; деление на 0
обработано.
**Зависимости:** math, numpy. **Предпосылки:** 6.  *(ε эффективности — пользовательский ввод/калибровка)*

## Задача 8 — Модель разрешения FWHM(E) 🟢 · Объём M
**Цель:** FWHM(E) детектора для поиска пиков, окна идентификации, деконволюции.

**Подзадачи:**
- 8.1. Сцинтилляционная модель `FWHM(E) = k·√(E + α·E²)` (AtomSpectra — сцинтиллятор), floor 0.01 keV².
- 8.2. Фит по реперным пикам (k, α) методом наименьших квадратов.
- 8.3. Bootstrap-провайдер: если репера нет — оценить FWHM по значимым пикам самого спектра.
- 8.4. Перевод FWHM(E)→FWHM(каналы) через калибровку для входа в поиск пиков.

**API (предлагается):**
```python
def fit_fwhm_scintillator(energies, fwhms_keV) -> FwhmModel   # .fwhm_at(E)->keV
def fwhm_provider(calibration, *, peaks=None) -> Callable[[int], float]  # каналы
```
**Источник:** `calibration/fwhm_fit.py` (`fit_fwhm_scintillator`, floor BUG-37);
`calibration/fwhm_provider.py` (иерархия + F-449 bootstrap).
**Тесты:** фит даёт монотонную FWHM(E); floor не даёт отрицательного; bootstrap без репера работает.
**Зависимости:** numpy. **Предпосылки:** 1 (и 2 для bootstrap).
