Краткий справочник: на чём проверено, какие проблемы ловятся чаще всего и что с ними
делать. Установка с нуля — в [`INSTALL.md`](INSTALL.md).

## 1. Совместимость окружения

### ✅ Проверено

| Компонент | Версия | Заметки |
|---|---|---|
| **Windows** | 11 Pro | Основная платформа разработки и прогона. |

| **Python** | 3.12 (64-bit) | Тоже работает; не смешивать в одном `.venv` с 3.14 (см. §2). |

| **PySide6 (Qt)** | 6.11.1 | Ставится как `cp310-abi3` wheel — валидно и на 3.14 (stable ABI). |
| **pyqtgraph** | 0.14.0 | 3D через `GLViewWidget` + `GLSurfacePlotItem`, 2D через `ImageItem` + `RectROI`. |
| **PyOpenGL** | 3.1.10 | Бэкенд OpenGL для 3D-вкладки. |
| **pytest** | 9.1.1 | 631 тестов, все зелёные. |

### ⚠ Требование к видео

3D-вкладка требует **OpenGL 2.1+**. На машинах без GL-драйвера (некоторые ВМ без
3D-ускорения, RDP-сессии) 3D-вкладка может быть чёрной или падать в инициализации GL.
2D-вкладка и слой данных от OpenGL не зависят.

## 2. Битый `.venv` после пересоздания другим Python ⚠ самая частая

**Симптом:** `ModuleNotFoundError: No module named 'numpy._core._multiarray_umath'`
(или похожий ImportError бинарного модуля) при запуске.

**Причина:** `python -m venv .venv` выполнен поверх существующего venv **другим**
интерпретатором (был 3.12 → пересоздали 3.14). `venv` подменяет `python.exe` и
`pyvenv.cfg`, но **не** переустанавливает пакеты. В итоге интерпретатор 3.14 пытается
загрузить скомпилированные под 3.12 `.pyd` — несовместимый ABI.

**Решение:** удалить `.venv` целиком и создать заново тем Python, которым будешь
запускать:
```powershell
Remove-Item -Recurse -Force .venv
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
```
Либо вообще не использовать venv — поставить зависимости глобально (`py -3.14 -m pip
install -r requirements.txt`); `run.bat` тогда запустит глобальным Python.

## 3. `run.bat`

| Проблема | Причина | Решение |
|---|---|---|
| «`run.bat` is not recognized…» | запуск не из своей папки | `cd` в папку проекта или запускать по полному пути; сам `.bat` делает `cd /d "%~dp0"` |
| Мусорные строки-команды в консоли (`'дпочитаем'…`) | редактор сохранил `.bat` в UTF-8, а cmd.exe читает его в OEM-кодировке (cp866) → кириллица в `REM` бьётся | комментарии в `run.bat` — **только ASCII** (так и сделано) |
| Открываются два окна | старый `run.bat` имел fallback `py -3.14 … \|\| python …`, запускавший оба | актуальный `run.bat` запускает ровно один процесс |

## 4. Образец данных не в репозитории

Реальные данные спектрометра в `.gitignore`: N42 (`*.n42`) может содержать GPS-координаты,
RadiaCode `*.rcspg` — deviceId/координаты/время, AtomSpectra `*.aswf` — serial и IP-источник.
Поэтому:

- тест загрузки образца помечен `skipif` и пропускается без файла;
- smoke-скрипты в `scripts/`, грузящие `sample_data/waterfall_sample.n42`, без образца
  работать не будут.

Положи свой `.n42` в `sample_data/waterfall_sample.n42`, чтобы прогнать их.

## 5. Ограничения v1 (по дизайну, не баги)

- **Нет спектроскопии:** поиск пиков, идентификация изотопов, калибровка по реперам,
  нетто/фон — отложены в v2 (см. README, «Дорожная карта»).
- **3D-рендер прорежается (LOD):** при больших матрицах поверхность даунсэмплится
  (≈400×512), чтобы держать FPS. Точные значения всегда доступны в 2D-карте и срезах —
  3D это обзорный вид, а не измерительный инструмент.
- **Цветовая шкала 2D — лог-контраст:** абсолютные отсчёты читаются в панели срезов и в
  подписи ROI («Сумма отсчётов = …»), не по цвету карты.
