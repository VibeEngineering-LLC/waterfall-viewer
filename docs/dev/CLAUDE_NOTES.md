## Конвенция замечаний (зафиксировано оператором 2026-06-26 — «Запомни навсегда»)

Дословно: «Фиксируй все замечания, присваивай номера. Отслеживай выполнение. Запомни навсегда».

1. **Фиксировать каждое замечание.** Любая правка/баг/пожелание из чата или со скриншота —
   записывается, не теряется. Источник истины — `TASKS.md` (раздел «Дефекты/Доработки …»).
2. **Присваивать сквозной номер** `#NN`. Нумерация непрерывна по всему проекту и **не сбрасывается**
   между фазами. Следующий свободный номер берётся за максимумом существующих в `TASKS.md`.
3. **Отслеживать выполнение** через список задач (TaskCreate/TaskUpdate): `pending` → `in_progress`
   → `completed`. Заголовок задачи начинается с `#NN — …`.
4. **При закрытии замечания** — дописать в `TASKS.md` запись `**#NN — суть** (файлы): что сделано`,
   обновить счётчик тестов («Полный pytest — N passed»), отметить задачу `completed`.
5. **Номер виден в коде:** в комментариях у изменённого кода ссылаться на `Задача #NN` —
   чтобы будущая сессия связала строку с замечанием.

`#137` — **образец и фон отображаются по-разному (образец шумный, фон сглажен) → вычет разный; усреднять образец и фон посегментно** (`awf/model/background.py` — новая `region_averaged_net`; `awf/ui/main_window.py` — переписан `_gated_for_3d` + импорт; `tests/test_background.py` +4 теста). Симптом оператора (2 скриншота окна среза): голубая кривая «образец» на выборке срезов [64:192] визуально шумнее оранжевой «фон»; «Вычитать можно только если способ отображения одинаков. Нужно определять на образце участки стабильности и усреднять — образец и фон внутри каждого участка отдельно». **Корень (systematic-debugging, Phase 1):** обе кривые используют один `smooth_counts`, но «образец» — это ПОДвыборка (меньше срезов → счётный шум √N на кривой), а «фон» — среднее по всему файлу (гладкое). Разная гладкость = «разный способ отображения» → и per-slice вычет несёт пуассонов шум, self-subtract не читается нулём без гейта #136. **Решение (выбор оператора через AskUserQuestion: «Режим усреднённый вычет»):** образец усредняется ТАК ЖЕ, как фон, но внутри каждого УЧАСТКА СТАБИЛЬНОСТИ (переиспользован `segment_by_time`, #131): в участке Ri `avg_smp_i[ch]=Σ_{t∈Ri}counts/Σ_{t∈Ri}lt` (cps/канал), `net_rate_i=avg_smp_i−bg_cps` (обе стороны — скорости, «одинаковый способ»), каждый срез t∈Ri получает `net_rate_i·lt[t]`. Применяется ТОЛЬКО при включённом «Вычет фона» (без вычета — данные как есть). Свойства (доказаны тестами): (1) образец и фон сглажены одинаково → гейт 3σ (#136) в этом режиме НЕ нужен и отключён; (2) интеграл нетто по каждому участку сохранён точно = Σ_{Ri}(counts−bg·lt) — знаковая модель среза #134 не нарушена; (3) при фон=весь файл и одном стационарном участке `avg_smp≡bg_cps ⇒ out≡0` поканально: самовычет одинаковых спектров = точный ноль (ровно «вычет одинаковых спектров = ноль» из BecqMoni, но честно — шум усреднён, а не погашен). `#139` — **в окне срезов кривая фона гладкая (усреднённый bg_cps) на фоне «лохматого» образца — сделать фон той же статистики** (`awf/model/background.py` — новая `background_window_like`; `awf/ui/panels.py` — `SlicePanel.set_background(bg_cps, raw)` + `_render_background`; `awf/ui/main_window.py` — стеш сырого фонового блока `self._bg_raw` в `_compute_background`; `tests/test_background.py` +5 тестов). Симптом оператора (скриншот окна срезов, 1D-кривые): голубая «образец» — лохматая (шумная), оранжевая «фон» — гладкая; «зачем усреднил фон? фон должен быть таким же лохматым как образец». **Корень (systematic-debugging, Phase 1, подтверждён чтением кода — НЕ угадан):** голубая «образец» = сырые суммарные отсчёты по короткому окну выборки срезов (напр. 128 срезов [64:192]) → полный пуассонов шум √N → лохматая; оранжевая «фон» = `bg_cps` (средняя скорость по ВСЕМУ фоновому диапазону, напр. 256 срезов) → шум подавлен √N → гладкая. `smooth_counts` применяется к обеим ОДИНАКОВО (один регулятор радиуса) — сглаживание НЕ причина; лохматость задаётся ЧИСЛОМ усреднённых срезов. **Решение (выбор оператора через AskUserQuestion: «Сырой фон, равный образцу по времени»):** оверлей фона в окне срезов строится как сырая сумма отсчётов фонового ПОДокна с ТЕМ ЖЕ живым временем, что у окна образца → совпадающий пуассонов шум. `background_window_like(bg_counts, bg_lt, target_lt)` суммирует сырые фоновые срезы (от начала) до накопления живого времени `target_lt` (= live-time окна образца), затем масштабирует ровно к `target_lt` (коэффициент ≈1) → та же статистика; при фоновый-диапазон=окно-образца кривые совпадают точно. Проводка: `_compute_background` для **range-источника** сташит сырой блок `self._bg_raw=(counts[lo:hi], lt[lo:hi])`, `set_background(bg, self._bg_raw)` передаёт его в `SlicePanel`; `_render_background` при наличии `_bg_raw` строит `background_window_like(cnt, lt, lt_total)` (в counts — как есть, в cps — /lt_total, единично с образцом), иначе падает на гладкий `bg_cps`. **file-источник → `_bg_raw=None`** (иная калибровка, сырой матч не строим — оверлей = гладкий `bg_cps`). **Область #139 — ТОЛЬКО 1D-оверлей окна срезов** (по скриншоту); эталон вычитания водопада (#134/#138) остаётся гладкой средней скоростью `bg_cps` (физически обязательно: один фон, много срезов — усреднение по диапазону корректно как эталон вычитания, шумить его нельзя). Тесты: `test_bg_window_full_range_equals_raw_sum` (полный диапазон = Σ сырых), `test_bg_window_scales_to_target_live_time` (масштаб к tgt), `test_bg_window_matches_sample_window` (фон=образец при совпадении окна → совпали), `test_bg_window_zero_target_returns_raw_sum`, `test_bg_window_bad_shape_raises`. **Полный pytest — 549 passed** (544 + 5). ## Ориентация по проекту

Десктоп-вьюер waterfall-спектрограмм гамма-спектрометра. Оси спектрограммы: **время × энергия**
(пространственной/GPS-оси в данных нет). Форматы: `.aswf`, `.rcspg`, `.n42`.

- **Стек:** Python 3.14, PySide6 (Qt), pyqtgraph 0.14.0, PyOpenGL (3D-waterfall), numpy, lxml.
- **Модель данных:** `awf/model/spectrogram.py` — `Spectrogram` (counts `[n_slices×n_channels]`,
  `time_offsets_s`/`real_time_s`/`live_time_s`, методы спектров/рядов/`downsample`/`counts_in_unit`)
  и `Calibration` (полином канал→энергия).
- **UI:** `awf/ui/` — `main_window.py` (окно, тулбар, доки, оркестрация сигналов), `view3d.py`
  (`Waterfall3DView`, GL-поверхность + секущие плоскости), `panels.py` (`HeatmapPanel` 2D-карта,
  `SlicePanel` спектр + временные ряды, `SectionControls`, `NuclidePanel`), `analytics_panel.py`,
  `style.py` (тёмная QSS-тема).
- **Цвета осей/плоскостей 3D:** время = бирюза `(51,217,242)`, энергия = пурпур `(242,89,217)`,
  счёт = жёлтый `(242,217,64)`. UI-элементы, привязанные к оси/плоскости, используют тот же цвет.
